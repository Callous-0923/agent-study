27.1 Agent Prompt 和普通 Prompt 的区别

 

普通 LLM Prompt：

只需要告诉模型「输出什么」

 

Agent Prompt：

需要告诉模型：

  • 你是谁（角色）
    • 2. 你能做什么（工具列表）
3. 什么时候该做什么（工具选择逻辑）
4. 什么不该做（边界/安全约束）
5. 输出格式是什么

     

    类比：

    普通 Prompt = 给厨师一个菜名

    Agent Prompt = 给厨师菜谱 + 厨房设备清单 + 工作流程

     

     

    27.2 Agent System Prompt 模板（面试送分题！）

     

    标准化模板（基于 Anthropic 2025 推荐）：

     

    
  你是一个 [角色描述]。


  你可以使用的工具：[工具列表]


  每个工具的使用场景和注意事项...


  解决问题时应该遵循的工作流程...


  你不能做的事情...


  输出格式要求...

     

    27.3 工具描述 = Prompt 工程 —— 本章核心！

     

    工具描述不是给开发者看的文档，是给 LLM 看的「使用说明」。

     

    好工具描述的三个要素：

  • 做什么：准确描述工具的功能
  • 什么时候用：描述适用的场景
  • 什么时候不用：描述边界条件

    •  

    对比（面试金句！）：

     

    ✗ 差： "查询天气"

    ✅ 好： "查询指定城市当天的实时天气，包括气温、湿度、风力。

          适用于用户询问天气相关问题时。不支持未来天气预报。
      示例调用：get_weather(city='北京')"

     

    工具描述评分卡：

      ┌────────────────────────┬──────────┐
  │ 描述要素                  │ 加分      │
  ├────────────────────────┼──────────┤
  │ 有功能描述                │ +1       │
  │ 有使用场景                │ +2       │
  │ 有边界条件                │ +2       │
  │ 有调用示例                │ +1       │
  │ 参数有 enum 限制          │ +2       │
  │ 参数有详细描述            │ +1       │
  └────────────────────────┴──────────┘

     

    评估工具描述的质量。

     

        Returns:
        评分详情。

     

    27.4 对话模板优化 —— 平衡信息密度

     

    Prompt 不是越长越好，需要权衡：

     

    太短 → 信息不足，LLM 靠猜测

    太长 → 信息过载，LLM 抓不住重点

     

    Golden Zone：

    System Prompt: 200-800 tokens

    工具描述: 每个工具 50-200 tokens

    总上下文: 不超过窗口的 70%（留空间给对话历史和生成）

     

    面试金句：

    "好 Prompt 的标准不是'全'，是'精'。

    每条信息都要有明确的 token 价值 vs 成本评估。"

     

     

    27.5 本章总结

     

    核心要点回顾：

     

  • Agent Prompt 需要结构化 6 个模块
  • 工具描述是 Agent Prompt 工程的核心
  • 好工具描述 = 做什么 + 什么时候用 + 什么时候不用 + 示例

    •  

    面试速记：

    → 核心：工具描述 = Prompt 工程（告诉 LLM 何时用、何时不用）

class AgentPromptBuilder:
    class=class="s">"s">class="s">""class="s">"Agent Prompt 构建器 —— 按模块组装，而非手写字符串。"class="s">""

    def __init__(self, role: str = class="s">"AI 助手"):
        self.role = role
        self.sections = {}

    def add_capabilities(self, tools: list[dict]):
        class=class="s">"s">class="s">""class="s">"添加工具能力声明。"class="s">""
        tool_lines = []
        for t in tools:
            tool_lines.append(fclass="s">"- {t[class='s'>'name']}: {t.get(class='s'>'description', class='s'>'')}")
        self.sections[class="s">"capabilities"] = class="s">"\n".join(tool_lines)

    def add_tool_guidance(self, guidance: dict[str, str]):
        class=class="s">"s">class="s">""class="s">"添加工具选择指引（什么时候用哪个工具）。"class="s">""
        lines = []
        for tool_name, guide in guidance.items():
            lines.append(fclass="s">"class="hclass="s">">### {tool_name}\n{guide}")
        self.sections[class="s">"tool_guidance"] = class="s">"\n\n".join(lines)

    def add_workflow(self, steps: list[str]):
        class=class="s">"s">class="s">""class="s">"添加工作流程。"class="s">""
        self.sections[class="s">"workflow"] = class="s">"\n".join(
            fclass="s">"{i+1}. {step}" for i, step in enumerate(steps)
        )

    def add_constraints(self, constraints: list[str]):
        class=class="s">"s">class="s">""class="s">"添加安全约束。"class="s">""
        self.sections[class="s">"constraints"] = class="s">"\n".join(
            fclass="s">"- {c}" for c in constraints
        )

    def add_output_format(self, fmt_desc: str):
        class=class="s">"s">class="s">""class="s">"添加输出格式。"class="s">""
        self.sections[class="s">"output_format"] = fmt_desc

    def build(self) -> str:
        class=class="s">"s">class="s">""class="s">"构建完整的 Agent System Prompt。"class="s">""
        parts = [class="s">"<system>"]
        parts.append(fclass="s">"  <role>{self.role}</role>")

        for key in [class="s">"capabilities", class="s">"tool_guidance", class="s">"workflow",
                     class="s">"constraints", class="s">"output_format"]:
            if key in self.sections:
                tag = key.replace(class="s">"_", class="s">"-")  class=class="s">"h"># tool_guidance → tool-guidance
                if tag == class="s">"capabilities":
                    tag = class="s">"capabilities"
                parts.append(fclass="s">"  <{tag}>")
                parts.append(self.sections[key])
                parts.append(fclass="s">"  </{tag}>")

        parts.append(class="s">"</system>")
        return class="s">"\n".join(parts)

    def estimate_tokens(self) -> int:
        class=class="s">"s">class="s">""class="s">"估算 Prompt 的 token 数。"class="s">""
        return len(self.build()) // 4

        return {
            class="s">"tool_name": tool.get(class="s">"name", class="s">"unknown"),
            class="s">"scores": scores,

        if pct >= 0.5:
            return class="s">"B (可用)"
        if pct >= 0.3:
            return class="s">"C (需改进)"
        return class="s">"D (不可用)"

    ]:
        print(fclass="s">"  {item}")
