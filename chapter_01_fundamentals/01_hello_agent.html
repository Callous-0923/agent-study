1.2 Function Calling —— Agent 的「手」

 

Function Calling 是 Agent 的核心机制。LLM 不只是生成文本，

它还能「决定」调用哪个函数、传什么参数。

 

流程：

用户：「北京今天天气怎么样？」

→

LLM 分析 → 识别需要调用 get_weather 函数

→

LLM 输出: {"name": "get_weather", "arguments": {"city": "北京"}}

→

我们的代码执行这个函数，拿到真实数据

→

把结果返回给 LLM，LLM 组织成自然语言回答用户

 

类比：LLM 像一位「盲人指挥家」，他能决定需要什么信息，

但必须由我们（代码）去执行实际的工具调用。

1.3 手写 Agent 循环 —— 理解底层原理

 

这是本章最核心的部分！

 

每个 Agent 的运行本质上是一个循环：

  • 把用户消息发给 LLM
  • LLM 判断：直接回答？还是需要调用工具？
  • 如果需要调用工具 → 执行工具 → 把结果发给 LLM，回到步骤 1
  • 如果可以回答 → 输出答案，结束

    •  

    这个循环被称为 ReAct 循环（Reasoning + Acting），

    是 LangChain / AutoGPT / crewAI 等所有框架的底层逻辑。

"k">def "f">run_agent(user_message: "b">str, max_iterations: "b">int = "n">5):
    "k">class="s">"""运行一个最简单的 ReAct Agent。

    这是不加任何框架的「裸写」Agent，帮助理解底层原理。
    任何 Agent 框架（LangChain/crewAI/AutoGen）本质上
    都是这个循环的封装 + 增强。

    Args:
        user_message: 用户输入的消息。
        max_iterations: 最大工具调用轮数，防止无限循环。

    Returns:
        Agent 的最终回答。
    """
    "k">class="h"># 对话历史 —— Agent 的「短期记忆」
    messages = [
        {"role": "system", "content": "你是一个有用的 AI 助手。当需要获取最新信息或执行计算时，请使用提供的工具。"},
        {"role": "user", "content": user_message},
    ]

    "b">print(f"\n{'='*">60}")
    "b">print(f"用户: {user_message}")
    "b">print(f"{'='*">60}")

    "k">for iteration "k">in "b">range(max_iterations):
        "k">class="h"># ===== 第 "n">1 步：调用 LLM（Think 阶段）=====
        model = os.getenv("LLM_MODEL", "gpt-4o-mini")
        response = client.chat.completions.create(
            model=model,
            messages=messages,
            tools=TOOLS,
            tool_choice="auto",
        )

        assistant_message = response.choices["n">0].message

        "k">class="h"># ===== 第 "n">2 步：检查 LLM 是否需要调用工具 =====
        "k">if assistant_message.tool_calls "k">is "k">None:
            "k">class="h"># LLM 决定直接回答（不需要工具）→ 循环结束
            final_answer = assistant_message.content
            "b">print(f"\n🎯 Agent 最终回答:\n{final_answer}")
            "k">return final_answer

        "k">class="h"># ===== 第 "n">3 步：执行工具调用（Act 阶段）=====
        "k">class="h"># 将 LLM 的回复（含 tool_calls）加入对话历史
        messages.append(assistant_message)

        "k">for tool_call "k">in assistant_message.tool_calls:
            func_name = tool_call.function.name
            func_args = json.loads(tool_call.function.arguments)

            "b">print(f"\n🔧 第 {iteration + ">1} 轮 - 调用工具: {func_name}({func_args})")

            "k">class="h"># 执行实际的工具函数
            tool_function = TOOL_MAP.get(func_name)
            "k">if tool_function "k">is "k">None:
                result = f"错误：未找到工具 {func_name}"
            "k">else:
                result = tool_function(**func_args)

            "b">print(f"📊 工具返回: {result}")

            "k">class="h"># 将工具执行结果加入对话历史（Observe 阶段）
            messages.append({
                "role": "tool",
                "tool_call_id": tool_call."b">id,
                "content": result,
            })

    "k">class="h"># 达到最大迭代次数，强制 LLM 给出最终回答
    "b">print("\n⚠️ 达到最大迭代次数，要求 LLM 给出最终回答...")
    messages.append({
        "role": "user",
        "content": "请基于已有的工具调用结果，给出最终回答。"
    })
    model = os.getenv("LLM_MODEL", "gpt-4o-mini")
    final_response = client.chat.completions.create(
        model=model,
        messages=messages,
    )
    final_answer = final_response.choices["n">0].message.content
    "b">print(f"\n🎯 Agent 最终回答:\n{final_answer}")
    "k">return final_answer
