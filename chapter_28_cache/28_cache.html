28.1 为什么 Agent 场景特别需要缓存？

 

Agent 的 LLM 调用特征：

  • 重复率高：用户问相似的问题（客服/教育场景 30-50% 重复）
  • 代价高：每次调用 = 金钱 + 时间
  • 步骤多：一个 Agent 任务可能调 5-10 次 LLM

    •  

    缓存收益估算：

    假设：日请求 10000 次，30% 可缓存，每次 $0.01

    日节省：10000 × 0.3 × $0.01 = $30/天

    年节省：$30 × 365 = $10,950/年

     

    加上延迟改善：缓存命中 P99 < 10ms vs LLM 调用 P99 > 2s)

     

     

    28.2 三级缓存架构

     

      ┌─────────────────────────────────────────────────┐
  │                 L1: Exact Match Cache              │
  │  ─────────────────────────────────               │
  │  查询: "今天天气" → 缓存中查 "今天天气"             │
  │  命中条件: 完全相同的文本                           │
  │  命中率: 5-10%                                    │
  │  延迟: <1ms                                       │
  ├─────────────────────────────────────────────────┤
  │                 L2: Semantic Cache                 │
  │  ─────────────────────────────────               │
  │  查询: "今天天气" → 缓存中查相似查询 → 命中 "今天天气怎么样"│
  │  命中条件: 余弦相似度 > 0.95                        │
  │  命中率: 20-40%                                    │
  │  延迟: ~10ms                                       │
  ├─────────────────────────────────────────────────┤
  │                 L3: LLM-as-Cache                   │
  │  ─────────────────────────────────               │
  │  查询: 无法在前2级命中 → 调用廉价小模型               │
  │  如果小模型回答合适 → 缓存结果                       │
  │  命中率: 10-20% (加上前两级：总 40-70%)              │
  │  延迟: ~500ms                                      │
  └─────────────────────────────────────────────────┘

    28.3 Token 预算管理系统

     

    类比：手机流量套餐

     

    每月 50GB：

  • 日常浏览（微信/网页）→ 低消耗
  • 看视频 → 高消耗
  • 快超标 → 限速 → 降级策略

    •  

    Agent Token 预算：

    每月 100M tokens：
  - 简单问答 → gpt-4o-mini (便宜)
  - 复杂分析 → gpt-4o (贵)
  - 快超标 → 只允许 mini + 摘要压缩

     

    Token 预算的 4 个层级：

  • 用户级：每人每月 N tokens
  • 会话级：每会话 N tokens
  • 请求级：每次 N tokens
  • 步骤级：每步 N tokens（防死循环）

    28.4 缓存的进阶策略

     

  • 分层 TTL

    • L1 精确缓存：永久（直到 LRU 淘汰）

    L2 语义缓存：24小时 TTL（信息会过时）

    L3 LLM 缓存：按查询类型设 TTL（新闻类 1h / 知识类 7天）

     

  • 缓存预热

    • 上线前用常见问题预填充缓存

     

  • 缓存失效

    • 知识库更新 → 相关缓存全部失效

     

  • 监控指标

    • 命中率 / 节省成本 / 缓存大小 / L1/L2/L3 分流比例
