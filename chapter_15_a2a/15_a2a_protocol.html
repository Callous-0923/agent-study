第15章：Google A2A 协议 —— Agent 之间的「世界语」

====================================================

 

 

 

Google 于 2025年4月9日发布 A2A 协议

联合 50+ 企业：Salesforce、SAP、MongoDB、LangChain...

 

 

15.1 A2A 解决了什么核心问题？

 

MCP 解决：「Agent 怎么和工具/数据库/API 交互？」

A2A 解决：「Agent 怎么和其他 Agent 交互？」

 

类比：

MCP = 手机和充电器之间的 USB-C 协议

A2A = 手机之间的通信协议（像 4G/5G）

 

对比 MCP vs A2A（面试必问！）：

 

  ┌──────────────┬─────────────────────┬─────────────────────┐
  │     维度      │        MCP           │        A2A           │
  ├──────────────┼─────────────────────┼─────────────────────┤
  │ 解决什么      │ Agent ↔ 工具/数据     │ Agent ↔ Agent       │
  │ 发布方        │ Anthropic (2024.11)  │ Google (2025.04)    │
  │ 架构          │ Client-Host-Server   │ Client-Server       │
  │ 核心概念      │ Tools/Resources/Prompts│ AgentCard/Task/Artifact│
  │ 通信协议      │ JSON-RPC 2.0         │ JSON-RPC 2.0 + HTTP │
  │ 传输方式      │ stdio / SSE          │ HTTP / SSE          │
  │ 典型场景      │ 查数据库/调API       │ 多Agent协作/委派任务 │
  └──────────────┴─────────────────────┴─────────────────────┘

 

配合使用的示例（面试可以这样描述）：

  • 用户 Agent 收到任务「预订去北京的出差行程」
  • 用户 Agent 通过 MCP 调用数据库 → 查用户偏好
  • 用户 Agent 通过 A2A → 发现「机票预订Agent」
  • 用户 Agent 通过 A2A → 将「订机票」子任务委派给机票Agent
  • 机票 Agent 通过 MCP → 调用航空公司 API
  • 机票 Agent 通过 A2A → 返回预订结果

    •  

     

    15.2 A2A 三大核心概念

     

  • AgentCard —— Agent 的「数字护照」

    •  

    每个 A2A Agent 都暴露一个 JSON 文件（/.well-known/agent.json），

    声明自己的能力、接口、安全要求：

     

    {

    "name": "TaxAgent",
"description": "税务计算和合规分析 Agent",
"url": "https://tax.example.com",
"capabilities": {
  "streaming": true,
  "pushNotifications": false
},
"skills": [
  {
    "id": "tax_calculation",
    "name": "税务计算",
    "description": "支持中美跨境税务计算",
    "inputModes": ["text", "file"],
    "outputModes": ["text", "file"]
  }
],
"defaultInputModes": ["text"],
"defaultOutputModes": ["text"],
"interfaces": [
  {"url": "https://tax.example.com/a2a", "transport": "JSONRPC"}
]

    }

     

  • Task —— A2A 的工作单元

    •  

    Task 是整个协议的核心抽象：

     

    生命周期：pending → working → input-required → completed/failed/cancelled

     

    Task 数据结构：

    {

    "id": "task_001",
"sessionId": "session_abc",
"status": {"state": "working", "message": "正在计算税务..."},
"artifacts": [...],
"history": [...]  // 操作历史

    }

     

  • Artifact —— 跨 Agent 的成果物

    •  

    Task 完成后的产出，携带 MIME 类型和元数据：

     

    {

    "artifactId": "report_001",
"name": "2025Q1税务报告.pdf",
"mimeType": "application/pdf",
"parts": [{"type": "file", "file": {...}}]

    }

     

     

    15.3 A2A 的完整交互流程

     

    Client Agent Server Agent

          │                                │
      │── GET /.well-known/agent.json ─→│  (1) 发现：获取 AgentCard
      │←── AgentCard ─────────────────│
      │                                │
      │── POST /a2a tasks/send ──────→│  (2) 提交任务
      │   {message: "请计算这笔税的...", │
      │    artifacts: [...]}           │
      │←── Task {status: "working"} ──│
      │                                │
      │── POST /a2a tasks/get ────────→│  (3) 查询进度（轮询）
      │←── Task {status: "working"} ──│
      │                                │
      │── POST /a2a tasks/get ────────→│
      │←── Task {status: "completed",  │  (4) 获取结果
      │    artifacts: [计算结果]}      │
      │                                │

     

    也可以使用 SSE 做流式推送（替代轮询）：

          │── POST /a2a (SSE) ───────────→│
      │←── event: status-update ──────│  (实时推送状态变化)
      │←── event: artifact-update ────│  (实时推送生成内容)
      │←── event: task-complete ──────│

      4. 取消任务（POST /a2a tasks/cancel）
    class="s">"""

    def __init__(self, agent_card: A2AAgentCard):
        self.agent_card = agent_card
        self.tasks = {}

    def get_agent_card(self) -> dict:
        class=class="s">"s">class="s">""class="s">"返回 AgentCard（Agent 发现阶段）。"class="s">""
        return self.agent_card.to_dict()

    def send_task(self, message: str,
                  session_id: str = None) -> A2ATask:
        class=class="s">"s">class="s">"""接收并开始执行一个任务。

        Args:
            message: 任务描述。
            session_id: 会话 ID（不传则自动生成）。

        Returns:
            创建的 Task 对象。
        class="s">"""
        task_id = hashlib.md5(
            fclass="s">"{message}-{time.time()}".encode()
        ).hexdigest()[:16]
        if session_id is None:
            session_id = hashlib.md5(
                fclass="s">"session-{time.time()}".encode()
            ).hexdigest()[:16]

        task = A2ATask(task_id, session_id)
        self.tasks[task_id] = task

        class=class="s">"h"># 模拟任务执行
        task.start()

        class=class="s">"h"># 找到匹配的 skill 并「执行」
        result_artifact = {
            class="s">"artifactId": fclass="s">"artifact_{task_id}",
            class="s">"name": fclass="s">"结果_{task_id[:8]}",
            class="s">"mimeType": class="s">"application/json",
            class="s">"parts": [{
                class="s">"type": class="s">"text",
                class="s">"text": json.dumps({
                    class="s">"summary": fclass="s">"已完成任务: {message[:50]}...",
                    class="s">"agent": self.agent_card.name,
                    class="s">"timestamp": time.time(),
                }, ensure_ascii=False),
            }],
        }
        task.complete([result_artifact])

        return task

    def get_task(self, task_id: str) -> Optional[A2ATask]:
        class=class="s">"s">class="s">"""查询任务状态。

        Args:
            task_id: 任务 ID。

        Returns:
            Task 对象，不存在返回 None。
        class="s">"""
        return self.tasks.get(task_id)

    def cancel_task(self, task_id: str) -> bool:
        class=class="s">"s">class="s">"""取消任务。

        Args:
            task_id: 任务 ID。

        Returns:
            是否成功取消。
        class="s">"""
        task = self.tasks.get(task_id)
        if task and task.state in (TaskState.PENDING, TaskState.WORKING):
            task.state = TaskState.CANCELLED
            task.status_message = class="s">"任务已被取消"
            return True
        return False


class A2AClient:
    class=class="s">"s">class="s">"""模拟 A2A Client —— 作为「用户 Agent」去调用其他 Agent。

    代表 Agent 完成：
      1. 发现远程 Agent（获取 AgentCard）
      2. 评估 skills（是否匹配任务需求）
      3. 提交任务 + 获取结果
    class="s">"""

    def __init__(self, name: str):
        self.name = name
        self.known_agents = {}  class=class="s">"h"># {url: AgentCard}

    def discover(self, server: SimulatedA2AServer) -> A2AAgentCard:
        class=class="s">"s">class="s">"""发现一个 A2A Agent —— 获取其 AgentCard。

        Args:
            server: A2A Server 实例。

        Returns:
            AgentCard 字典。
        class="s">"""
        card = server.get_agent_card()
        self.known_agents[card[class="s">"url"]] = card
        return card

    def find_agent_for_skill(self, skill_keyword: str) -> Optional[dict]:
        class=class="s">"s">class="s">"""根据需求关键词找到匹配的 Agent。

        Args:
            skill_keyword: 技能关键词。

        Returns:
            匹配的 AgentCard，未找到返回 None。
        class="s">"""
        for _, card in self.known_agents.items():
            for skill in card.get(class="s">"skills", []):
                if skill_keyword in skill.get(class="s">"name", class="s">"").lower() or \
                   skill_keyword in skill.get(class="s">"description", class="s">"").lower():
                    return card
        return None

    def delegate_task(self, server: SimulatedA2AServer,
                      message: str) -> dict:
        class=class="s">"s">class="s">"""将任务委派给远程 Agent。

        完整的 A2A 委派流程：
          Discover → Evaluate → Send → Wait → Receive

        Args:
            server: A2A Server 实例。
            message: 任务描述。

        Returns:
            包含任务结果的字典。
        class="s">"""
        result = {}

        class=class="s">"h"># Step 1: 发现
        card = self.discover(server)
        result[class="s">"agent"] = card[class="s">"name"]
        result[class="s">"skills"] = [s[class="s">"name"] for s in card[class="s">"skills"]]

        class=class="s">"h"># Step 2: 提交任务
        task = server.send_task(message)
        result[class="s">"task_id"] = task.id
        result[class="s">"task_state"] = task.state.value

        class=class="s">"h"># Step 3: 获取结果
        final_task = server.get_task(task.id)
        result[class="s">"final_state"] = final_task.state.value
        result[class="s">"artifacts"] = final_task.artifacts

        return result
