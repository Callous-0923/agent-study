=================================

 

 

 

 

Agent 类型全景图

 

  ┌────────────────────────────────────────────────────┐
  │                                                    │
  │  1. ReAct Agent        → 每步推理，最通用          │
  │  2. Plan-Execute Agent → 先规划再执行，结构化任务   │
  │  3. Reflexion Agent    → 执行+反思，自我改进       │
  │  4. Tool-calling Agent → 纯工具调用，不需要推理     │
  │  5. Multi-Agent        → 多角色协作（第5章详讲）    │
  │                                                    │
  └────────────────────────────────────────────────────┘

 

各类型关键区别：

┌──────────────┬──────────┬──────────┬──────────┬──────────┐
│     特性      │  ReAct   │Plan-Exec │Reflexion │Tool-Call │
├──────────────┼──────────┼──────────┼──────────┼──────────┤
│ 规划方式      │ 动态推理  │ 前置规划  │ 动态+反思 │ 不规划   │
│ 复杂任务      │ 中等     │ 强        │ 强        │ 弱       │
│ 执行效率      │ 中等     │ 高        │ 低(多次)  │ 高       │
│ 错误恢复      │ 自动     │ 需重新规划 │ 强(反思)  │ 弱       │
│ Token 消耗    │ 中等     │ 低        │ 高        │ 低       │
│ 典型场景      │ 通用     │ 数据分析  │ 代码审查  │ API 网关 │
└──────────────┴──────────┴──────────┴──────────┴──────────┘

 

面试速记：

"请对比 ReAct 和 Plan-Execute"

→ ReAct 灵活但费 token，Plan-Execute 高效但不灵活

→ 大型任务用 Plan-Execute 做顶层规划，ReAct 做每步执行

→ Google 的 ReAct 论文是 Agent 领域最重要论文之一

3.1 ReAct Agent（面试必问！）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ReAct = Reasoning + Acting（推理 + 行动）

执行流程：
  Thought → Action → Observation → Thought → Action → ... → Final Answer

每次循环：
  1. Thought: class="s">"我需要知道 X" → 推理
  2. Action: 调用 search(class="s">"X") → 行动
  3. Observation: 收到搜索结果 → 观察
  → 基于观察继续思考，直到能给出最终答案

类比：
  就像你在解决一道复杂数学题——
  你不可能一次算出结果，而是：
  class="s">"先化简...然后代入...发现不对...换个思路..."
  每一次class="s">"想 → 做 → 看结果"就是一个 ReAct 循环。

论文来源：
  Yao et al., class="s">"ReAct: Synergizing Reasoning and Acting in Language Models"
  (ICLR 2023) — Agent 领域的奠基性论文
class="s">"""


class ReActAgent:
    class=class="s">"s">class="s">"""ReAct Agent 实现 —— 最经典的 Agent 模式。

    核心算法：
      while not done:
          response = llm(messages + tools)
          if tool_calls:
              execute_tools(response.tool_calls)
              append results to messages
          else:
              return response.content
    class="s">"""

    def __init__(self, system_prompt: Optional[str] = None):
        class=class="s">"s">class="s">"""初始化 ReAct Agent。

        Args:
            system_prompt: 自定义系统提示词。
        class="s">"""
        self.system_prompt = system_prompt or (
            class="s">"你是一个有用的助手。面对需要实时信息或计算的问题，"
            class="s">"请使用提供的工具。推理过程请在思考后行动。"
        )

    def run(self, user_message: str, max_steps: int = 5) -> str:
        class=class="s">"s">class="s">"""执行 ReAct 循环。

        Args:
            user_message: 用户消息。
            max_steps: 最大步数限制。

        Returns:
            Agent 的最终回答。
        class="s">"""
        messages = [
            {class="s">"role": class="s">"system", class="s">"content": self.system_prompt},
            {class="s">"role": class="s">"user", class="s">"content": user_message},
        ]

        for step in range(max_steps):
            print(fclass="s">"\n  [ReAct 第 {step+1} 步]")

            response = client.chat.completions.create(
                model=MODEL, messages=messages,
                tools=TOOLS, tool_choice=class="s">"auto",
            )
            msg = response.choices[0].message

            if msg.tool_calls is None:
                print(fclass="s">"  → 最终回答")
                return msg.content

            messages.append(msg)
            for tc in msg.tool_calls:
                name = tc.function.name
                args = json.loads(tc.function.arguments)
                print(fclass="s">"  → 调用: {name}({args})")
                result = TOOL_MAP[name](**args)
                messages.append({
                    class="s">"role": class="s">"tool",
                    class="s">"tool_call_id": tc.id,
                    class="s">"content": result,
                })

        return class="s">"达到最大步数限制，无法完成任务。"

3.2 Plan-Execute Agent
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

「先规划，再执行」—— 像项目经理一样工作。

执行流程：
  1. 分析任务 → 制定详细执行计划
  2. 按计划逐步执行每个子任务
  3. 汇总各步骤结果 → 生成最终答案

适用场景：
  - 任务步骤明确、有依赖关系
  - 数据分析报表生成
  - 代码审查流水线
  - 工作流自动化
class="s">"""


class PlanExecuteAgent:
    class=class="s">"s">class="s">""class="s">"Plan-Execute Agent —— 先规划再逐步执行。"class="s">""

    def __init__(self):
        self.plan = []
        self.results = {}

    def _make_plan(self, task: str) -> list:
        class=class="s">"s">class="s">"""调用 LLM 制定执行计划。

        Args:
            task: 用户任务描述。

        Returns:
            计划步骤列表。
        class="s">"""
        prompt = (
            fclass="s">"请将以下任务分解为 2-4 个可执行的子步骤。"
            fclass="s">"返回 JSON: {{\"steps\": [{{\"id\": 1, \"desc\": \"...\", "
            fclass="s">"\"tool\": \"search|calculator|none\", "
            fclass="s">"\"tool_input\": \"...\"}}]}}\n\n任务: {task}"
        )
        response = client.chat.completions.create(
            model=MODEL,
            messages=[{class="s">"role": class="s">"user", class="s">"content": prompt}],
            response_format={class="s">"type": class="s">"json_object"},
            temperature=0.3,
        )
        data = json.loads(response.choices[0].message.content)
        return data.get(class="s">"steps", [])

    def run(self, task: str) -> str:
        class=class="s">"s">class="s">"""执行 Plan-Execute 流程。

        Args:
            task: 用户任务描述。

        Returns:
            执行结果汇总。
        class="s">"""
        print(class="s">"\n  [Plan-Execute] 阶段 1: 制定计划")
        self.plan = self._make_plan(task)

        for step in self.plan:
            print(fclass="s">"    步骤 {step[class='s'>'id']}: {step[class='s'>'desc']}")
            print(fclass="s">"      工具: {step[class='s'>'tool']}")

        print(class="s">"\n  [Plan-Execute] 阶段 2: 执行计划")
        for step in self.plan:
            tool = step.get(class="s">"tool", class="s">"none")
            tool_input = step.get(class="s">"tool_input", class="s">"")
            if tool == class="s">"none":
                result = fclass="s">"已完成: {step[class='s'>'desc']}"
            elif tool in TOOL_MAP:
                result = TOOL_MAP[tool](tool_input)
            else:
                result = fclass="s">"暂不支持的工具: {tool}"
            self.results[fclass="s">"step_{step[class='s'>'id']}"] = result
            print(fclass="s">"    步骤 {step[class='s'>'id']} 结果: {result}")

        print(class="s">"\n  [Plan-Execute] 阶段 3: 生成汇总")
        summary_prompt = (
            fclass="s">"任务: {task}\n"
            fclass="s">"执行结果:\n{json.dumps(self.results, ensure_ascii=False)}\n"
            fclass="s">"请基于以上结果给出最终回答。"
        )
        response = client.chat.completions.create(
            model=MODEL,
            messages=[{class="s">"role": class="s">"user", class="s">"content": summary_prompt}],
        )
        return response.choices[0].message.content

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
3.3 Reflexion Agent
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

「做一遍 → 反思 → 改进 → 再做一遍」—— 自我迭代优化。

执行流程：
  1. 初次执行任务
  2. 自我评估：打分 + 找问题
  3. 基于反馈重新执行
  4. 重复直到满意

核心洞察：
  Reflexion 不改变 Agent 的基础能力，
  而是通过「反馈循环」让 Agent 在实践中变得更好。
  就像学生学习：做题 → 对答案 → 发现错在哪里 → 下次改进。

论文来源：
  Shinn et al., class="s">"Reflexion: Language Agents with Verbal Reinforcement Learning"
  (NeurIPS 2023)
class="s">"""


class ReflexionAgent:
    class=class="s">"s">class="s">""class="s">"Reflexion Agent —— 执行 + 自我反思 + 迭代改进。"class="s">""

    def __init__(self, max_reflections: int = 3):
        class=class="s">"s">class="s">"""
        Args:
            max_reflections: 最大反思迭代次数。
        class="s">"""
        self.max_reflections = max_reflections

    def _evaluate(self, answer: str, task: str) -> dict:
        class=class="s">"s">class="s">"""自我评估回答质量。

        Args:
            answer: Agent 的回答。
            task: 原始任务。

        Returns:
            包含评分和问题的字典。
        class="s">"""
        prompt = (
            fclass="s">"任务: {task}\n"
            fclass="s">"回答: {answer}\n\n"
            fclass="s">"请从准确性、完整性、清晰度三方面评分(1-10)，"
            fclass="s">"并指出具体问题。返回 JSON:\n"
            fclass='s'>'{{class="s">"accuracy": 0, class="s">"completeness": 0, class="s">"clarity": 0, '
            fclass='s'>'class="s">"issues": [class="s">"问题1"]}}'
        )
        response = client.chat.completions.create(
            model=MODEL,
            messages=[{class="s">"role": class="s">"user", class="s">"content": prompt}],
            response_format={class="s">"type": class="s">"json_object"},
        )
        return json.loads(response.choices[0].message.content)

    def run(self, task: str) -> str:
        class=class="s">"s">class="s">"""执行 Reflexion 流程。

        Args:
            task: 用户任务描述。

        Returns:
            经过反思迭代后的最佳回答。
        class="s">"""
        print(fclass="s">"\n  [Reflexion] 任务: {task}")
        feedback_history = []

        for iteration in range(self.max_reflections):
            print(fclass="s">"\n  [Reflexion] 第 {iteration+1} 轮")

            class=class="s">"h"># 构建带反馈的提示词
            if feedback_history:
                feedback_text = class="s">"\n".join(
                    fclass="s">"上一轮问题 {i+1}: {fb}"
                    for i, fb in enumerate(feedback_history[-1])
                )
                prompt = (
                    fclass="s">"任务: {task}\n"
                    fclass="s">"请改进之前的回答。之前的问题:\n{feedback_text}"
                )
            else:
                prompt = task

            class=class="s">"h"># 执行（不调用工具，纯推理）
            response = client.chat.completions.create(
                model=MODEL,
                messages=[{class="s">"role": class="s">"user", class="s">"content": prompt}],
            )
            answer = response.choices[0].message.content
            print(fclass="s">"    回答: {answer[:100]}...")

            class=class="s">"h"># 自我评估
            evaluation = self._evaluate(answer, task)
            avg_score = (
                evaluation.get(class="s">"accuracy", 0)
                + evaluation.get(class="s">"completeness", 0)
                + evaluation.get(class="s">"clarity", 0)
            ) / 3
            issues = evaluation.get(class="s">"issues", [])
            feedback_history.append(issues)
            print(fclass="s">"    评分: {avg_score:.1f}/10 | 问题数: {len(issues)}")

            class=class="s">"h"># 质量达标则停止
            if avg_score >= 8.0:
                print(fclass="s">"    质量达标，停止迭代！")
                return answer

        return answer

    class="s">"react": ReActAgent,
    class="s">"plan_execute": PlanExecuteAgent,
    class="s">"reflexion": ReflexionAgent,
}


def run_agent_by_type(agent_type: str, task: str):
    class=class="s">"s">class="s">"""统一的 Agent 运行入口。

    Args:
        agent_type: Agent 类型（react / plan_execute / reflexion）。
        task: 用户任务。

    Returns:
        Agent 的执行结果。
    class="s">"""
    agent_class = AGENT_REGISTRY.get(agent_type)
    if agent_class is None:
        return fclass="s">"不支持的 Agent 类型: {agent_type}。支持: {list(AGENT_REGISTRY.keys())}"

    print(fclass="s">"\n{class='s'>'='*60}")
    print(fclass="s">"  Agent 类型: {agent_type.upper()}")
    print(fclass="s">"  任务: {task}")
    print(fclass="s">"{class='s'>'='*60}")

    agent = agent_class()
    result = agent.run(task)
    print(fclass="s">"\n  ✅ 最终答案:\n{result}")
    return result

   - 最通用、最灵活
   - 缺点：每步都调用 LLM，token 消耗大

   - 先规划后执行
   - 适合结构化任务

   - 适合质量要求高的任务
   - 缺点：多次 LLC 调用，成本高

4. 实际项目中往往是组合使用
   - 高层 Plan-Execute 做任务分解
   - 每层 ReAct 做具体执行
   - 关键步骤加 Reflexion 做质量把关
